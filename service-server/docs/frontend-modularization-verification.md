# 프론트엔드 모듈화 수동 검증

## 자동 검증

```bash
npm run test:frontend
GRADLE_USER_HOME=/tmp/deepchess-gradle ./gradlew test
```

프런트엔드 테스트는 평가 계산, 게임 트리와 탐색, 분석 큐의 순차 실행·부모 대기·재시도·초기화를 검증한다.

## 수동 검증 준비

1. Python 체스 엔진과 Spring Boot 서비스 서버를 실행한다.
2. Google OAuth로 로그인하고 닉네임 설정을 완료한다.
3. 브라우저 개발자 도구의 Console과 Network 탭을 연다.

## 분석 보드

- [ ] 메인 화면에서 `새 게임 분석`을 선택하면 `/board.html`로 이동하고 새 게임이 생성된다.
- [ ] 체스말을 드래그하면 합법적인 도착 칸이 표시되고 드래그 종료 후 표시가 제거된다.
- [ ] 불가능한 수는 원래 칸으로 snapback 된다.
- [ ] 합법적인 수를 두면 기보와 현재 수 강조가 갱신된다.
- [ ] 첫 분석 요청이 성공하고 평가치와 추천 수가 표시된다.
- [ ] 여러 수를 연속으로 두어도 `/api/analysis` 요청은 하나씩 순서대로 처리된다.
- [ ] 자식 분석 요청의 `parentPositionId`는 부모 분석 응답 이후에만 전송된다.
- [ ] 일반 평가 점수와 `+M3`, `M3`, `-M2` 같은 메이트 점수가 평가치 바에 맞게 표시된다.
- [ ] 상위 추천 수가 최대 3개 표시된다.
- [ ] 기보의 수를 클릭하면 해당 FEN으로 보드가 이동한다.
- [ ] 같은 부모에서 다른 수를 두면 variation이 들여쓰기되어 표시된다.
- [ ] 처음·이전·다음·마지막 버튼이 메인 라인을 탐색한다.
- [ ] 입력창에 포커스가 없을 때 방향키가 같은 방식으로 탐색한다.
- [ ] 보드 회전 버튼이 정상 동작한다.

## FEN과 PGN 가져오기

- [ ] 메인 화면에서 `기보 불러오기`를 선택하면 불러오기 영역만 표시된다.
- [ ] 올바른 FEN을 입력하면 현재 국면이 로드되고 저장·분석이 시작된다.
- [ ] 올바른 PGN을 입력하면 전체 수순과 게임 결과가 로드된다.
- [ ] 잘못된 FEN/PGN은 경고 후 전송되지 않는다.
- [ ] 가져온 수순에서 분석되지 않은 노드가 순차 분석된다.

## 저장과 보관함

- [ ] 저장 버튼은 새 게임 또는 가져온 게임에서만 표시된다.
- [ ] 저장 성공 후 버튼이 `저장 완료` 상태로 바뀐다.
- [ ] `/mypage.html`에서 저장된 게임 목록과 날짜, 미리보기가 표시된다.
- [ ] 목록 항목을 클릭하거나 키보드로 활성화하면 기존 `?gameId=` URL로 이동한다.
- [ ] 저장된 게임 트리가 다시 로드되고 분석 데이터가 복원된다.
- [ ] 분석이 없는 저장 노드만 다시 분석된다.

## 반응형 화면

- [ ] 1350px 이하에서 주요 패널이 자연스럽게 줄바꿈된다.
- [ ] 600px 이하에서 보드, 기보, 제어 패널이 한 열로 표시되고 조작 가능하다.
